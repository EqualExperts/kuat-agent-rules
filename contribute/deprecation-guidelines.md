# Deprecation guidelines

> **Status: TBD — Phase 5 fast-follow.**

How Kuat retires a component, pattern, asset, or skill — the deprecation path (status +
replacement + migration note) and how the change is communicated to the network.

These guidelines will be **designed against the first real case: retiring the legacy
`kuat-create` / `kuat-review` skills in Phase 5.** Designing the process against a concrete
retirement (rather than in the abstract) keeps it grounded. Until then, raise any deprecation in
Slack **[#design-system](https://equalexperts.slack.com/archives/C0BCFBB4EK0)** and treat it as a
custodian-led **Heavy** change.

See the [model overview](./overview.md) §7 for the release/communicate/deprecate posture.
