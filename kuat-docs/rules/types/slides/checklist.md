# Slides — Output Checklist

Run before delivering a deck.

**See also:** [README.md](./README.md), [brand-compliance.md](./brand-compliance.md), [layouts.md](./layouts.md).

---

- [ ] Title slide carries deck title + subtitle + EE logo + hero photograph in B&W
- [ ] Section eyebrow + slide title structure used on every content slide
- [ ] No body copy below 11pt
- [ ] All photographs B&W
- [ ] One layout per slide — no hybrids
- [ ] Stats panels paired with context, not numbers alone
- [ ] Page number badge present on body slides; absent on title, dividers, full-bleed
- [ ] Closing slide with named contact, photograph, email, phone, social row
- [ ] Sub-brand / co-brand follows endorsement pattern, not equal weight
- [ ] One chart per slide, conclusion in the title
- [ ] Case-study slides are named, true, and (for external use) approved
- [ ] Voice is active, claims are unhedged, jargon is stripped
- [ ] Layout count is small (5–8 across the deck), not one-per-slide

If any item fails and cannot be reconciled with the user's request, flag the conflict in your output rather than silently breaking the rule.
