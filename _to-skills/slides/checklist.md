# Slides — Output Checklist

Use for **pre-delivery (create)** and **compliance review (review)**. See skill [kuat-review](../../../../skills/kuat-review/SKILL.md) for review intake and report format.

**See also:** [README.md](./README.md), [brand-compliance.md](./brand-compliance.md), [layouts.md](./layouts.md).

---

## Checklist

- [ ] Title slide carries deck title + subtitle + EE logo + hero photograph in B&W
- [ ] Section eyebrow + slide title structure used on every content slide
- [ ] No body copy below 11pt
- [ ] All photographs B&W
- [ ] One layout per slide — no hybrids
- [ ] Stats panels paired with context, not numbers alone
- [ ] Page number badge present on body slides; absent on title, dividers, full-bleed
- [ ] Closing slide with named contact, photograph, email, phone, social row
- [ ] Sub-brand / co-brand follows endorsement pattern, not equal weight
- [ ] One chart per slide, conclusion in the title
- [ ] Case-study slides are named, true, and (for external use) approved
- [ ] Voice is active, claims are unhedged, jargon is stripped
- [ ] Layout count is small (5–8 across the deck), not one-per-slide

---

## When reviewing — evidence to look for

| Item | Evidence |
|------|----------|
| Title slide | Opening slide shows title, subtitle, B&W hero, EE logo placement |
| Section structure | Content slides use eyebrow + title pattern |
| Type size | Measure or inspect body text ≥ 11pt |
| Photography | All photos monochrome |
| Layouts | Each slide matches one layout from [layouts.md](./layouts.md); no mixed templates |
| Stats | Numbers appear with explanatory copy on same slide |
| Page badge | Body slides: blue badge top-right; absent on title/divider/full-bleed |
| Closing | Final slide has contact block per [brand-compliance.md](./brand-compliance.md) |
| Co-brand | Client logo not equal weight to EE on cover |
| Charts | Single chart per slide; title states the conclusion |
| Case studies | Named client; claims defensible; approval noted if external |
| Voice | Active voice; no hedge chains; jargon reduced |
| Layout variety | Count distinct layouts; target 5–8 per deck |

If any item fails and cannot be reconciled with the user's request, flag the conflict in your output rather than silently breaking the rule.
