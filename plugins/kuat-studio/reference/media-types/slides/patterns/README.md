# Slide Patterns

Deck-type patterns — content density, formality, and slide budget differ by deck type.

| Pattern | For |
|---------|-----|
| [sales-marketing.md](./sales-marketing.md) | Pitch decks, proposals, client-facing capability decks |
| [knowledge-sharing.md](./knowledge-sharing.md) | Internal talks, conferences, communities of practice |
| [case-studies.md](./case-studies.md) | Named client stories (standalone or embedded) |
| [reporting.md](./reporting.md) | Status reports, research findings, audits (read-ahead) |

> **Pattern model (later cut):** These deck-type patterns will adopt the
> [outcome-framed pattern model](../../../patterns/overview.md), linking shared concepts where a deck
> type is also built in another medium. [case-studies.md](./case-studies.md) already links up to its
> shared concept ([describe a case study](../../../patterns/help-users/describe-a-case-study.md)); the
> rest stay as-is for now.
